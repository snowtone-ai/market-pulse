# /escape — エラー連鎖脱出 Skill

## トリガー
同じエラーが2回以上解決しない / 同じアプローチを繰り返している / 袋小路に入った感覚

## 実行手順

1. `.claude/issues.md` を読む
2. 現在のアプローチとエラーの要約を `.claude/escape.md` に書き込む:

```markdown
## Escape記録 — [日付]

### 行き詰まったアプローチ
[何をやろうとして何が起きたか]

### 試みた対策（全て失敗）
1. [対策1]
2. [対策2]

### 根本原因の仮説
[なぜ解決しないかの考察]

### 代替アプローチ候補
1. [別のアプローチ]
2. [さらに別のアプローチ]
```

3. ユーザーに以下を伝える:
```
/escape を記録しました。
Opus 4.7 で別アプローチを検討するため、/clear を実行してください。
新しいセッションで「/escape の内容を確認して別アプローチを提案してください」と伝えてください。
```

4. それ以上の実装を停止する（止まることが正しい）
